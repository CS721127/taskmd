# 配置与存储后端（Configuration & Storage Backends）

## 1. 配置原则（Configuration Principles）

配置应支持以下优先级：

1. CLI 参数（CLI Args）
2. 环境变量（Environment Variables）
3. 配置文件（Config File）
4. 默认值（Defaults）

## 2. 默认配置目录（Default Config Directory）

建议：

- Linux/macOS: `~/.config/taskmd/`
- Windows: `%APPDATA%/taskmd/`

## 3. 示例配置（Example Config）

```toml
db_path = "~/.config/taskmd/tasks.md"
theme = "dracula"
editor = "code"
timezone = "Australia/Sydney"
live_reload = true
auto_git_backup = false
storage_backend = "local"
```

## 4. 存储后端（Storage Backends）

### 第一优先（Recommended First）
- `LocalFileBackend`
- `SyncedFolderBackend`（OneDrive / iCloud / Dropbox 同步目录）

### 后续（Later）
- `SSHBackend` / `SFTPBackend`
- `WebDAVBackend`
- `CustomBackend`

## 5. 存储抽象（Storage Abstraction）

所有 backend 应至少支持：
- read_text
- safe_write_text
- exists
- stat / mtime
- optional lock / conflict handling

## 6. 数据安全（Data Safety）

- 原子写入（Atomic Write）
- 备份（Backup）
- 可选 Git 历史（Optional Git History）
- 冲突处理基础（Conflict Handling Groundwork）
