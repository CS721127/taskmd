# 架构总览与分层（Architecture Overview & Layers）

## 1. 总体分层（High-Level Layers）

TaskMD 推荐划分为以下层级：

1. **CLI / UI Layer**：输入输出与终端渲染
2. **Command Layer**：命令解析与路由
3. **Service Layer**：业务逻辑与用例
4. **Repository / Storage Layer**：读写、备份、锁、后端选择
5. **Parser / Writer Layer**：Markdown / Sidecar / Legacy TXT 解析与写回
6. **Data Layer**：`tasks.md` 与 `.taskmd/items/*.md`

## 2. 架构目标（Architecture Goals）

- 能支持主任务文件和 Sidecar
- 能支持本地与远程存储后端
- 能支持 Live Reload 与 Low-Diff Writeback
- 能隔离 Connector / AI 等高变动能力
- 能测试 parser / service / cli / ui snapshot

## 3. 数据流（Data Flow）

### 读路径（Read Path）
`Backend -> Repository -> Parser -> Models -> Services -> CLI/UI`

### 写路径（Write Path）
`CLI/UI -> Commands -> Services -> Repository -> Writer -> Backend`

### Sidecar 路径（Sidecar Path）
`Task ID -> Sidecar Service -> .taskmd/items/<id>.md -> Resource/Action Resolution`

## 4. 极后期扩展位（Late-Stage Extension Slots）

- Connector Registry
- Plugin API
- AI Provider Layer
- Remote Storage Backend Layer
- Optional Hotkey / Desktop Companion
