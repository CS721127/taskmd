# 仓库结构与模块职责（Repository Layout & Module Responsibilities）

## 1. 推荐仓库结构（Recommended Repository Layout）

```text
TaskMD/
├─ pyproject.toml
├─ README.md
├─ LICENSE
├─ CHANGELOG.md
├─ docs/
├─ examples/
├─ src/taskmd/
├─ tests/
└─ assets/
```

## 2. `src/taskmd/` 模块结构（Package Layout）

```text
src/taskmd/
├─ app.py
├─ cli.py
├─ config.py
├─ exceptions.py
├─ models/
├─ parser/
├─ storage/
├─ services/
├─ commands/
├─ ui/
├─ integrations/
├─ plugins/
└─ utils/
```

## 3. 模块职责（Responsibilities）

### models/
定义 Task、Metadata、Sidecar、Section 等实体。

### parser/
解析 Markdown / Sidecar / Legacy TXT，负责 schema 约束。

### storage/
负责 backend、repository、backup、locking。

### services/
任务操作、查询、风险、进度、统计、导出、Sidecar、Action、AI。

### commands/
CLI 子命令入口。

### ui/
terminal / rich / dashboard / themes。

### integrations/
editor、browser、pdf、calendar、maps、finance、weather、course 等连接器。

### plugins/
极后期插件注册与 hook。

## 4. 文档与测试（Docs & Tests）

设计文档、架构文档、路线图、CLI 规范。

### tests/
- parser/
- roundtrip/
- services/
