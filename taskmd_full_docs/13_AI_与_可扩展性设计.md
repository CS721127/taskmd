# AI 与可扩展性设计（AI & Extensibility Design）

## 1. AI 的定位（Positioning of AI）

AI 不是核心存储逻辑的一部分，而是一个 **可选智能辅助层（Optional Intelligence Layer）**。

## 2. AI 建议适合做的事情

- 任务整理（Task Grooming）
- 优先级建议（Priority Suggestion）
- 周报 / 总结生成（Report Generation）
- 自然语言命令解释（NL Command Interpretation）
- 行动建议（Action Suggestions）

## 3. AI 边界（AI Boundaries）

- 默认关闭（Opt-in）
- 用户自己配 API / endpoint / key
- 所有写操作应确认（Confirm before Mutation）
- AI 不能绕过主文件真相源（Source of Truth）

## 4. 可扩展性设计（Extensibility）

### Extension Points
- custom openers
- custom exporters
- custom connectors
- AI provider adapters
- plugin hooks

### 为什么要这样设计
让项目在未来可以生长，而不是不断堆 if/else。
