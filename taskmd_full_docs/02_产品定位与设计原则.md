# 产品定位与设计原则（Positioning & Design Principles）

## 1. 产品定位（Positioning）

TaskMD 的定位是：

> **一个以 Markdown 为真相源（Source of Truth）、以 CLI 为增强交互层、允许编辑器直接编辑、并可逐步扩展到任务执行入口与连接器生态的本地任务系统。**

### 与传统任务工具的区别

- **不是黑盒数据库（Not a Black-Box Database）**
- **不是纯 GUI First（Not GUI-First）**
- **不是只能在 App 中编辑（Not App-Locked）**
- **不是单纯 todo.txt 克隆（Not Just a todo.txt clone）**

## 2. 核心设计原则（Core Principles）

### 2.1 文件即数据库（The File Is the Database）
主任务文件本身应该具备长期价值。

### 2.2 手工编辑是一等公民（Manual Editing Is First-Class）
手工编辑不是“兼容”，而是正式工作流。

### 2.3 低扰动写回（Low-Diff Writeback）
写回时尽量保留结构、空行和说明文本。

### 2.4 复杂上下文外置（Move Rich Context to Sidecar）
主文件保持简洁，复杂资源 / 动作 / 上下文放到 Sidecar。

### 2.5 可扩展但不早耦合（Extensible, Not Prematurely Coupled）
地图、课程平台、AI、财务连接器都应通过接口实现。

### 2.6 先稳后炫（Infrastructure Before Magic）
Schema、Parser、Writer、Round-trip Tests、CLI、Storage 必须先做好。

## 3. 用户体验原则（UX Principles）

- 命令短、语义清晰（Short & Clear Commands）
- 输出紧凑但不混乱（Compact but Understandable Output）
- 文件可直接读懂（Human-Readable Files）
- 可选主题与视觉增强，但不牺牲信息准确性
- 不强迫用户理解底层元数据和存储实现
