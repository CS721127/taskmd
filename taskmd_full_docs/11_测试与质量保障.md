# 测试与质量保障（Testing & Quality Assurance）

## 1. 测试目标（Testing Goals）

TaskMD 的测试核心不是“覆盖率数字”，而是保证：
- 数据不丢（No Data Loss）
- 手工编辑不易写坏（Manual Editing Robustness）
- Schema 稳定（Schema Stability）
- 导出与命令行为可预测（Predictable CLI / Export Behavior）

## 2. 必测模块（Must-Test Areas）

### Parser Tests
- 标题与任务解析
- metadata 解析
- 非法 metadata 容错

### Writer Tests
- 低扰动写回
- 缺失 ID 自动补全
- 非任务文本保留

### Round-trip Tests
- read -> write -> read
- edit externally -> reload -> save -> reload

### Service Tests
- progress
- heatmap
- risk
- recurring
- timestamp

### CLI Tests
- add / done / due / rm / list / open / validate

### Export Tests
- ICS / PDF / CSV / JSON / HTML / image (where possible)

## 3. Snapshot Tests
Rich 输出可以用 snapshot 测试，避免 UI 不经意退化。

## 4. 数据安全措施（Safety Measures）

- safe write
- backup
- validation command
- optional Git backup
- conflict groundwork for remote backends
