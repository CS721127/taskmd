# 导出与集成设计（Export & Integration Design）

## 1. Export Layer 的目标（Goals）

让 TaskMD 的数据可以流入：
- 日历（Calendar）
- 打印版计划（Printable Plans）
- 汇报 / 共享材料（Reports / Sharing）
- 二次分析脚本（Analysis Pipelines）

## 2. 导出类型（Export Types）

### ICS
将 due 任务转成日历事件。

### PDF Monthly
把某月任务排成月历视图。

### CSV / JSON
适合分析和外部处理。

### Image
把 dashboard 或摘要渲染为长图。

### HTML Board
把任务导出为本地 HTML 看板。

## 3. Integrations 设计原则（Integration Principles）

- 作为 adapters / connectors 存在
- 不把 provider-specific 逻辑写死进 core
- 敏感配置放 config，不放 tasks.md
- 失败时不应破坏核心任务数据
