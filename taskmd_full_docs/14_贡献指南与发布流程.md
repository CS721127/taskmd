# 贡献指南与发布流程（Contribution & Release Workflow）

## 1. 贡献者指南（Contributor Guidance）

建议在仓库中准备：
- `CONTRIBUTING.md`
- issue templates
- PR template
- code style notes
- testing instructions

## 2. Pull Request 基本要求

- 说明动机（Motivation）
- 描述影响范围（Impact Scope）
- 附测试或说明为什么不需要测试
- UI 变更附截图 / snapshot
- 涉及 schema / CLI 的变更必须同步文档

## 3. Release 建议

- 使用 Semantic Versioning
- 保留 `CHANGELOG.md`
- 重要 schema 变化必须附 migration notes

## 4. GitHub Actions 建议

- lint
- test
- build
- docs check
- release publish
